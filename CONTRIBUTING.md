# Contributing

KUN is proprietary source-available software. Contributions are welcome only
under the terms below.

## Contribution Terms

By submitting an issue, pull request, patch, comment, design, test case,
benchmark, prompt, workflow, idea, or other contribution, you confirm that:

1. you have the legal right to submit it;
2. it does not contain confidential material you are not allowed to share;
3. it does not include code copied from another project unless that code is
   clearly identified and license-compatible;
4. you grant the project owner a perpetual, worldwide, royalty-free,
   irrevocable, sublicensable license to use, reproduce, modify, distribute,
   commercialize, and otherwise exploit the contribution as part of KUN or
   related products.

No employment, partnership, payment, ownership, or future license is implied by
submitting a contribution.

## What We Accept

- bug reports;
- security reports through the private security channel described in
  `SECURITY.md`;
- small fixes that match the existing architecture;
- tests that improve honesty, safety, delivery status, or tenant isolation;
- documentation fixes that do not reveal secrets or private user data.

## What We Do Not Accept

- code or documents that weaken proprietary notices;
- contributions that add open-source license headers to KUN-owned files;
- copied third-party code without clear provenance;
- secrets, credentials, user data, private logs, or customer data;
- changes that claim unfinished capabilities are complete.

## Development Rule

KUN has a strict honesty rule: do not mark a capability as `ready` unless there
is runnable evidence. Partial work must stay labeled as partial.

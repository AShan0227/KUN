operate, benchmark, sell, host, train, demo, or support a commercial or internal
business product without prior written permission from the project owner.

- code and APIs;

- Watchtower / NUO / Qi / Hermes designs;
- task routing, MoE strategy, state ledger, memory, and self-evolution logic;
- UI/UX flows, dashboards, and task-board concepts;
- test fixtures, benchmark fixtures, dogfood scenarios, and red-team scenarios.

## Allowed Without Written Permission

Only these limited actions are allowed:

1. viewing the public repository on GitHub;
2. opening issues or pull requests for review;
3. referencing small, non-substantial excerpts for security reporting or code
   review, with clear attribution and without commercial reuse.

### 方式 A: Web UI (最直观)

> ⚠️ 当前前端只是 walking skeleton. 主对话 UI 在, 管家面板可能是占位. 跑 `kun doctor` 看后端 health.

### 方式 B: CLI (最简单, 1 行)

```bash

# 看启状态
kun qi status

# 看 5 个 seed 协议

# 看某个协议详情
kun protocol get writing.creative.short --version 0.1.0-seed

# 给一个虚拟任务找匹配协议
```

### 方式 C: HTTP API

```bash

  -H "Content-Type: application/json" \

# WebSocket: 用浏览器 console 或 wscat
```

## 4. 验"V2.3 真在工作" — 看后端 log

| `protocol.applied protocol_id=writing.creative.short applied_mode=SMART` | ✅ 协议真消费了, ExecutionMode 被改 |

| `qi_darwin.protocol_emerged` | 🎉 涌现新 experimental protocol! |

跑完跑 `kun protocol list` 应看到 5 starter + 涌现的 `darwin.emergent.*`.

## 7. 故障排查

### "kun: command not found"
```bash
cd /Users/petrarain/KUN
./scripts/install_kun_cli.sh
# 然后 source ~/.zshrc 或新开终端
```

### "uv run kun: No such file or directory"
你不在 KUN 项目目录. 要么 cd 到 KUN, 要么装 wrapper (上一条).

### "前端 npm run dev 报 EADDRINUSE 3000"
你旧 KUN 项目的前端进程还在跑. 两选一:
```bash
# A: kill 旧进程 (PID 看 lsof)
lsof -i :3000     # 找 next-server PID
kill <PID>

# B: 用别的端口 (推荐, 不动旧进程)
PORT=3001 npm run dev   # 默认就是 3001 了
PORT=3002 npm run dev   # 也行
```

### "后端 kun serve 报 port 8000 in use"
```bash
kun serve --port 8001
```

### "kun doctor 报 alembic head 不在 0016_pheromone"
```bash
cd /Users/petrarain/KUN
uv run alembic upgrade head
```

### "Darwin 跑了但没涌现 protocol"
score < 0.6 阈值. 看 log:
```bash
grep "qi_darwin.done" <log>
```
LLM 回答太短 (length_bonus 算法). 多跑几次, 或调阈值.

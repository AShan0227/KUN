- 主对话框: `app/page.tsx` — 输入消息, 看流式回复

### 方式 B: CLI (最简单, 1 行)

```bash
# 直接跑 1 个 task
uv run kun run "为新产品写一段 30 字 slogan"
```

## 4. 验"V2.3 真在工作"

跑下面任意 1 个 task, 后端 log 应看到这些事件:

### 验 1: 协议消费 (KUN_PROTOCOL_CONSUME_ENABLED=1, default ON)
跑 `uv run kun run "为新产品写一段 30 字 slogan"`, 后端 log 应看到:
```
protocol.applied  protocol_id=writing.creative.short  applied_mode=SMART
```
说明 orchestrator 真的在消费协议, 改了 ExecutionMode.

### 验 2: 反作弊检测 (KUN_ANTI_GAMING_ENABLED=1, default ON)
跑 `uv run kun run "回答: 1+1=?"`, 如果 LLM 偷懒 (比如 copy_prompt), log 应看到:
```
gaming.detected  pattern=copy_prompt  severity=high
```
(正常情况下不会触发, 这是反作弊兜底)

### 验 3: Pheromone 涌现 (default ON)
跑 3-4 个不同 task, 每个 task 完后 log 应看到 step 完成. 跑 `kun protocol list` 不会变 (因为 Pheromone 是 skill chain 级, 不是 protocol 级). 但启窗口里 Darwin 跑探索后会涌现新 protocol.

### 验 4: 启窗口里 Darwin 跑探索 (default ON, 但需"真在窗口里")
两种触发:
1. **等 cron**: 默认 hourly, 但要时间窗口内 (默认 1-5 AM)
2. **强制触发**: `export KUN_QI_FORCE_ACTIVE=1` 然后等下一个整点 (cron 是 @hourly)

或直接手动跑一次:
```bash
KUN_QI_ENABLED=1 KUN_QI_FORCE_ACTIVE=1 uv run python -c "
import asyncio
from types import SimpleNamespace
from starlette.datastructures import State
from kun.api.runtime import install_runtime
from kun.qi.cron_jobs import _qi_darwin_godel_explore
from kun.watchtower.engine import RuleEngine

app = SimpleNamespace(state=State())
install_runtime(app, rule_engine=RuleEngine([]))
asyncio.run(_qi_darwin_godel_explore(app, 'u-sylvan'))
"
# log 应看到: qi_darwin.done rounds=3 best_score=0.X
# 如果 score >= 0.6 → qi_darwin.protocol_emerged 一个新 experimental protocol
# 跑完后再 `kun protocol list` 应多 1 个 darwin.emergent.*
```

## 6. 如果你看不到东西怎么办

### "启了 server 但 protocol list 是空的"
```bash
# 检查 seed 是否跑了
grep "protocol_seed.done" <后端 log>
# 如果没看到, 手动跑:
KUN_PROTOCOL_SEED_DEFAULTS=1 uv run python -c "
import asyncio
from kun.qi import get_protocol_registry
from kun.qi.seed_protocols import seed_default_protocols
asyncio.run(seed_default_protocols(get_protocol_registry()))
"
```

### "前端 npm run dev 跑不起来"
```bash
cd frontend
rm -rf node_modules package-lock.json
npm install
npm run dev
```

### "CLI 报 import error"
```bash
# 重新装依赖
uv sync --extra dev
# 重跑
uv run kun version  # → 1.x.x
```

### "Darwin 跑了但没涌现 protocol"
```bash
# best_score 没到 0.6 阈值. 看 log:
grep "qi_darwin.done" <log>
# 如果 best_score < 0.6, 说明 LLM 回答太短 (length_bonus 算法). 
# 可调阈值或多跑几次.
```

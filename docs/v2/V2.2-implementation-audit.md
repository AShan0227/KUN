## 给 V2.3 讨论的起点

V2.2 接近收尾 (~88%). 用户将整理 V2.3 方向. 几个值得考虑的:

1. **从 spec 驱动 → 从 user dogfood 驱动**: V2.2 是按 spec 章节实装. V2.3 应该跑真 task → 看哪里卡 → 反推 wire. 这是从"为产品造功能" → "为用户解决问题".

2. **ENSEMBLE 真上线**: V2.2 §21 有 ENSEMBLE 设计但还没真接 (codex C32). 上线后是高 stakes task 的真正质变 — 同 task 跑 5 path multi_judge 选最优, 给用户 winner + "5 个方案对比".

3. **mempalace 真用**: GraphTraversal (Wire 30) 通了, 但 entity_relationships 表数据来源主要靠 RelationshipMineStep daily 跑 events log. 如果 events log 数据稀疏, 图就空. V2.3 可以加更多 relation 写入点 (e.g. 用户显式标注 / LLM 解释时带 relations).

4. **真 V2.3 spec 来自 dogfood 反馈**: 跑 1-2 周真 task → 收集 lab 推过来的 recipe + classifier 决策日志 + value_gate intervention → 看模式 → 写 V2.3.
